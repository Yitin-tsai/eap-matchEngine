# Investigation Summary: Timed Double Auction

## 機制定義

每小時啟動一場 10 分鐘密封式集合競價（Sealed-Bid Call Auction），以 Uniform Price 清算，處置未來一小時的電力交割。與現有連續雙邊拍賣（CDA）並存，用戶自行選擇交易模式。

## 時間線

```
:00 ─── 拍賣開啟（收集下一小時交割的出價）
:10 ─── Gate Closure（固定截止，無隨機偏移）
:10~:11 ── 系統清算（計算 MCP）
:11 ─── 結果公佈 + 錢包結算
:00（下一小時）── 電力交割
```

## 確認的設計決策

| # | 項目 | 決策 |
|---|------|------|
| 1 | 截止時間 | 固定 :10 分整，不使用隨機截止 |
| 2 | 定價規則 | 成交量最大化 → 多解時取中間價 |
| 3 | 邊際訂單分配 | Pro-rata 按比例分配 |
| 4 | 出價格式 | 階梯式出價（stepwise bid curve） |
| 5 | 負價格 | 不支援（price >= 0） |
| 6 | 價格上下限 | 管理員可配置（非寫死） |
| 7 | 拍賣期間可見資訊 | 僅顯示參與人數 |
| 8 | 與 CDA 關係 | 並存，用戶選擇交易模式 |

## 清算演算法

1. 收集所有買單（按價格降序排列 → 需求曲線）
2. 收集所有賣單（按價格升序排列 → 供給曲線）
3. 找到供需曲線交叉點 → MCP（Market Clearing Price）+ MCV（Market Clearing Volume）
4. 所有成交者統一以 MCP 結算
5. 邊界情況：
   - 不交叉 → 流標，無成交
   - 多解 → 取成交量最大的價格，仍有多解取中間價
   - 邊際訂單 → pro-rata 按比例分配

## 出價格式（階梯式）

```json
{
  "userId": "uuid",
  "side": "BUY",
  "auctionId": "AUC-2026040910",
  "steps": [
    { "price": 60, "amount": 50 },
    { "price": 55, "amount": 30 },
    { "price": 50, "amount": 20 }
  ]
}
```

- 出價後不可修改、不可刪除
- 密封：截止前不可見其他人出價，僅公開參與人數

## 結算流程

- 買方：鎖定 = sum(step.price × step.amount)；成交以 MCP 結算，差額退還
- 賣方：鎖定 = sum(step.amount)；成交以 MCP 結算
- 未成交部分：全額解鎖
- 部分成交（邊際訂單）：依 pro-rata 比例成交，剩餘解鎖

## 與現有 CDA 的差異

| 面向 | CDA（現有） | Call Auction（新增） |
|------|------------|---------------------|
| 撮合時機 | 即時 | 批次（gate closure 後） |
| 定價 | 逐筆撮合價 | 統一清算價 MCP |
| Orderbook | 公開即時推送 | 密封，僅顯示參與人數 |
| 出價格式 | 單點 price+amount | 階梯式 steps[] |
| 修改/取消 | 可取消 | 不可 |
